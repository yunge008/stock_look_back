# stock_look_back｜本地股票 / ETF 回测

一个不依赖 TradingView 的中文量化研究工具。它用 AkShare 获取 A 股和 ETF 日线，用 FastAPI 执行日频策略回测，用 React + Plotly 展示参数、权益曲线、交易流水和逐笔 lot，SQLite 保存每次回测。

> 回测不构成投资建议。个股基本面质量筛选目前由用户确认；后续接入财务数据时必须按公告日对齐，禁止前视。

## 快速使用

1. 输入 A 股、ETF 或美股代码，例如 `603728`、`600519`、`513500`、`AAPL`、`BRK.B`、`00700`、`09988`。
2. 代码右侧会自动显示证券简称；名称查询失败不会影响回测。
3. 在主面板设置日期与四个核心低位条件，点击“开始回测”。
4. 需要调整资金、补仓、止盈、费用或质量确认时，打开“参数设置”TAB。
5. 左侧“历史基本面选股器”按所选日期筛选 A 股（不含 ETF）；“一揽子标的网格组合回测”支持股票与 ETF 分别运行同一套质量网格参数。

## 默认参数（质量过滤逐笔网格）

- 已通过质量筛选：默认勾选；
- 策略资金池：200,000 元；
- 首仓金额：30,000 元；
- 回撤回看：360 个交易日；
- MA：120 日；
- 最高收盘回撤：30%；
- 低于 MA：15%；
- 入场条件：同时满足；
- 补仓支持 3–10 层；默认五层跌幅为 10% / 20% / 30% / 40% / 50%，倍数为 1x / 2x / 2x / 3x / 3x；
- 策略资金池上限默认 20 万元；建议首仓金额按 `策略资金池上限 ÷（1 + 各层补仓倍数之和）` 计算且可修改，修改首仓金额、补仓层数或倍数时会同步调整资金池上限；
- 单 lot 止盈：10%；默认每持仓满 365 个自然日，止盈要求降低 1 个百分点，最低为 0%；默认关闭组合清仓，再入场回撤：0%；
- 佣金率：5%，最低佣金 5 元，卖出税费率 0.1%，单边滑点 5%。所有百分比框直接输入百分数（输入 `5` 即表示 `5%`），输入框右侧固定显示 `%`。

A 股或关闭碎股的 ETF，若默认订单金额不足一手且资金池可覆盖，会自动提高至一手实际所需资金（含费用）；资金池仍为硬上限，不会透支或融资。

SQLite 数据库和行情缓存默认保存在 %LOCALAPPDATA%\stock_look_back，避免 OneDrive 离线文件或同步锁导致启动时出现 disk I/O error。可通过 DATABASE_PATH 和 DATA_CACHE_DIR 环境变量覆盖。

投入资金年化回报率按期末总盈亏与最大实际资金占用估算，年化分母只累计有未平仓 lot 的持仓交易日，空仓等待期不计入，统一按每年 245 个交易日折算；它不等同于总账户 CAGR，需结合最大资金占用一并查看。Lot 持仓时间按买入成交日至卖出成交日的自然日差计算；未平仓 Lot 截止回测最后交易日。 质量过滤逐笔网格的回撤图先按当日未平仓 Lot 的总成本计算持仓收益率，再相对该收益率的历史高点计算回撤，因此只显示 0 或负数；空仓日不计入，不受闲置现金和策略资金池上限影响。 权益图同时显示策略净值（时间加权、剔除场外现金流）；账户内买卖不视为外部资金。

## 目标买点

目标买点按最新可用交易日计算：`min(过去 N 日最高收盘 × (1 − 回撤阈值), MA × (1 − MA 偏离阈值))`。只有收盘价不高于该较低价格时，最高回撤与 MA 下方两个条件才会同时满足。输入有效代码后页面会自动测算；调整核心参数后可点击按钮立即重算。页面同时显示两条单独阈值、最新收盘价和与目标的距离。

## 策略口径

- 所有信号使用当日收盘价计算，并在下一交易日开盘成交；最后一天的信号不会被强制成交。
- 每笔买入都是独立 lot；达到该 lot 实际成本 + 止盈阈值后，全量卖出，不拆单。
- 补仓只锚定本轮首仓的实际成交价，不用平均成本。
- 组合清仓只关闭剩余 lot；清仓后默认需满足相对最终卖出价的再入场回撤和低位条件。再入场跌幅设为 `0` 时取消价格回撤限制，但仍须满足低位条件。
- 回测到期默认按最后交易日收盘价强制平仓；剩余 lot 的盈亏进入已实现盈亏和真实胜率。

## 历史基本面选股器与组合回测

- 选股器以 A 股历史业绩报表为股票池，按原始公告日期 `NOTICE_DATE` 截断所选日期之后的数据，明确排除 ETF；后续修订日期 `UPDATE_DATE` 不再冒充首次公告日。
- 默认条件对应截图：市值约 20 亿美元、PE(TTM) 0–30、净利率/ROE/营收同比/每股收益同比均不低于 5%，252 个交易日高点回撤 30%，低于 SMA120 15%。高点回看天数、回撤比例、SMA 周期和偏离比例均可修改。
- 历史分析师评级与严格稀释每股收益没有可靠统一快照，页面会明确提示，绝不以当前数据或近似值静默替代。
- 组合回测支持 A 股、ETF、美股和港股代码，按总金额等分；每个标的独立运行与本地回测一致的质量网格参数，不采用一次买入持有。组合层汇总实际并发资金、现金、持仓市值、持仓比例和权益。
- 组合结果并列输出含现金总权益回撤与按股票持仓金额计算的回撤；收益图中的持仓金额、现金金额默认隐藏，持仓比例默认显示。相关系数已取消，改为按标的筛选查看完整 Lot 明细。
- 组合回测可选沪深300、纳斯达克100、标普500、中证1000、上证50、科创50或创业板指作为参考基准；基准按首个可交易日归一到组合初始金额，并输出基准总收益、245交易日年化、最大回撤及组合年化超额。
- 所有年化指标按每年 245 个交易日折算；自然日只用于显示持仓时长和网格止盈递减周期。
## 行情与缓存

- A 股：东方财富前复权优先，失败时回退新浪前复权日线。
- ETF：东方财富前复权优先，失败时回退新浪未复权日线，并在页面提示风险。
- 美股：Yahoo Finance 日线，使用自动复权 OHLC（包括拆股与分红调整）。
- 港股：输入 5 位纯数字代码（如 `00700`、`09988`）；优先使用 Yahoo Finance 自动复权 OHLC，失败时依次回退东方财富、新浪前复权日线。港股整手数因标的而异，当前回测按 1 股最小单位模拟。
- 历史行情和数据源元数据保存在 `backend/data_cache/`。
- 新标的历史不足时，回测会自动改用实际可用的最早开始日期，并将回撤回看与 MA 周期缩短至可用交易日最大值；页面和回测警告会显示实际采用的参数。
- 数据源全部失败时会返回明确错误，绝不生成伪造回测结果。

## 本地启动

需要 Python 3.11+、Node.js 20+。

```powershell
cd "E:\OneDrive\1. AI\股票基金回测\backend"
py -3.11 -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
Copy-Item .env.example .env
python main.py
```

另开 PowerShell：

```powershell
cd "E:\OneDrive\1. AI\股票基金回测\frontend"
npm install
npm run dev
```

前端：<http://localhost:5173>；API 文档：<http://127.0.0.1:8000/docs>。

## 使用方式

本项目仅在本地运行：FastAPI 通过本机 Python 调用 AkShare，React 页面通过本机浏览器访问。源码同步到 GitHub；需要使用时克隆或拉取最新代码，按上面的“本地启动”步骤启动前后端。项目不依赖 Vercel、Lovable 或公网后端。
